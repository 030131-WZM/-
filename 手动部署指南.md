## 📋 手动部署步骤

### 步骤1：打开命令提示符
1. 按 `Win + R` 
2. 输入 `cmd` 并回车
3. 或者搜索"命令提示符"

### 步骤2：进入部署目录
```cmd
cd C:\Users\90760\.openclaw\workspaceopenclaw config\deploy
```

### 步骤3：推送代码到GitHub
```cmd
git push -u origin main
```

### 步骤4：输入GitHub认证信息
如果弹出认证窗口，请：
1. 输入你的GitHub用户名
2. 输入你的GitHub个人访问令牌（推荐）或密码

### 步骤5：启用GitHub Pages
1. 访问你的GitHub仓库：https://github.com/030131-WZM/-
2. 点击 "Settings"
3. 找到 "Pages" 选项（在左侧菜单）
4. 在 "Source" 下选择 "Deploy from a branch"
5. Branch选择 "main"
6. 点击 "Save"

### 步骤6：等待部署完成
通常需要1-5分钟，页面会显示部署状态。

## 🔑 GitHub认证说明

### 方法1：使用个人访问令牌（推荐）
1. 访问：https://github.com/settings/tokens
2. 点击 "Generate new token"
3. 选择 "Generate new token (classic)"
4. 设置token名称和过期时间
5. 勾选 `repo` 权限
6. 点击 "Generate token"
7. 复制生成的token（只显示一次）

### 方法2：使用GitHub CLI
```cmd
gh auth login
```

## 📞 遇到问题？

### 问题1：认证失败
- 检查用户名和密码是否正确
- 确保有仓库的推送权限
- 尝试使用个人访问令牌

### 问题2：推送失败
- 检查网络连接
- 确保仓库地址正确
- 删除远程仓库重新添加：`git remote remove origin`

### 问题3：GitHub Pages不显示
- 确保代码已推送成功
- 等待5-10分钟
- 检查仓库是否为Public
